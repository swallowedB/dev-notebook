## 큐브 좌표계
- 3개의 축 x, y, z를 사용해서 위치를 표현하는 방법
  
### 원리
- `x + y + z = 0` 이라는 조건을 만족해야함

## 1) 큐브 좌표계 말고 다른 좌표계? 차이와 선택 이유
- 큐브 좌표계가 방향 계산이 제일 직관적이고 인접한 셀 계산이 쉽기 때문에 큐브 좌표계를 선택
- 사실상 지금 쓰고 있는건 Cube -> Axial로 변환
- 계산은 큐브로 하되, 월드 좌표 변환은 2D로 한느 패턴

| 좌표계              | 표기                       | 특징/장점                        | 단점/언제 피하나                   |
| ---------------- | ------------------------ | ---------------------------- | --------------------------- |
| **픽셀 좌표**        | (x, y) 화면 픽셀             | 렌더링/마우스 좌표와 1:1              | 이웃/거리/링 계산이 복잡, 격자 수학 불편    |
| **오프셋(Offset)**  | (col, row) + odd/even 규칙 | 2D 배열처럼 저장/순회 쉬움, 직관적 저장구조   | 이웃 계산이 홀/짝에 따라 달라져 로직 분기 많음 |
| **더블드(Doubled)** | (q, r) but 간격 2          | 좌표가 격자선에 딱 맞아 떨어져 변환 간단      | 값이 커짐, 보편성이 떨어짐             |
| **축(Axial)**     | (q, r)                   | 2D로 간결, 대부분 연산(이웃/회전/선/링) 깔끔 | 거리 계산 등에 변환(→ 큐브) 필요        |
| **큐브(Cube)**     | (x, y, z), **x+y+z=0**   | 이웃/거리/라인/링 공식이 가장 단순·대칭      | 값 3개 관리(메모리↑), 최종 렌더엔 변환 필요 |

### 🔍 큐브 좌표계가 뭔가요?
- 육각 격자를 수학적으로 안정적으로 표현하기 위한 3축 좌표계 입니다.
- 이 좌표계는 항상 x, y, z축을 사용하고 모두 합하여 0이 되도록 하는 조건을 만족시켜야 합니다.
- 큐브 좌표계의 장점은 육각형 간의 이웃 탐색, 거리 계산, 링 확장 같은 연산을 모두 대칭적이고 단순한 공식으로 구현할 수 있다는 점입니다.
